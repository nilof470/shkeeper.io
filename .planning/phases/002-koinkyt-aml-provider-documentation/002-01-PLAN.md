<threat_model>
Trust boundaries:
- SHKeeper to Koinkyt HTTPS API boundary.
- Operator secret boundary for `KOINKYT_API_KEY` inside `aml-shkeeper`.
- SHKeeper to `grither-pay` callback boundary carrying AML decisions.

STRIDE register:
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-02-01 | API key leaks through docs, logs, or pasted probe responses | docs and live probes | Docs must instruct using env vars and sanitized responses; never commit or paste real key values. |
| T-02-02 | Unsupported assets are auto-credited because Koinkyt coverage is incomplete | coverage docs and implementation | Docs must require fail-closed manual review for assets outside btc/eth/trx native, USDT, USDC. |
| T-02-03 | Provider error or missing risk score is mistaken for approval | failure policy | Docs must define incomplete/error responses as manual review or retry-until-timeout, never credit. |
| T-02-04 | Token transfer endpoint is misused and checks the wrong counterparty | token check docs | Docs must mark `/transfer` as unresolved until live probes confirm whether `/transaction` is sufficient. |
</threat_model>

<threat_model>
Trust boundaries:
- SHKeeper to `aml-shkeeper` HTTP Basic Auth boundary.
- `aml-shkeeper` to AMLBot external provider boundary.
- `aml-shkeeper` database boundary storing raw provider evidence.

STRIDE register:
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-01-01 | Spoofed check creation | `POST /api/v1/checks` | Reuse existing Basic Auth check and reject missing required identity fields. |
| T-01-02 | Duplicate checks cause conflicting AML states | check creation | Enforce `deposit_id` and `idempotency_key` uniqueness and return existing state. |
| T-01-03 | Provider errors are mistaken for approval | task normalization | Missing score, provider error, or timeout must return non-success provider status and no auto-approved state. |
| T-01-04 | Sensitive AMLBot credentials leak | config/logging | Do not log `access_key`, `access_id`, generated token, or request payload containing secrets. |
</threat_model>

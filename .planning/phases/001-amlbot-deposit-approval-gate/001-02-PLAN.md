<threat_model>
Trust boundaries:
- SHKeeper database boundary storing local AML snapshots.
- SHKeeper to `aml-shkeeper` HTTP Basic Auth boundary.
- Merchant callback consumer boundary where grither-pay relies on normalized decision values.

STRIDE register:
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-02-01 | AML fields are attached to the wrong deposit | `AmlCheck.transaction_id` | Enforce unique foreign key to `Transaction.id`. |
| T-02-02 | Missing coverage mapping auto-credits deposits | `aml_coverage.py` | Fail closed to `manual_review` and add coverage test over `Crypto.instances`. |
| T-02-03 | Small skipped deposits appear as zero-risk AML checks | `aml_policy.py` | Store `score=None` and `status="skipped"` for skips. |
| T-02-04 | Sidecar transport errors become approvals | `aml_shkeeper_client.py` | Client errors return non-terminal/error state; policy resolves to `manual_review` only through explicit reason. |
</threat_model>

  <files>shkeeper/services/aml_policy.py, tests/test_aml_policy.py</files>

  <action>
Create `shkeeper/services/aml_policy.py` with class or functions that make policy decisions using persisted `Transaction` and `AmlCheck` data. Required public functions: `build_deposit_id(tx)`, `build_idempotency_key(tx)`, `should_skip_aml(tx)`, `build_skipped_check(tx)`, `decision_from_provider_result(tx, result)`, and `is_terminal(aml_check)`.

`build_deposit_id(tx)` must return `shkeeper-tx-{tx.id}`. `build_idempotency_key(tx)` must return `{tx.crypto}:{tx.txid}:shkeeper-tx-{tx.id}`.

`should_skip_aml(tx)` must compare `tx.amount_fiat` against `AML_MIN_CHECK_AMOUNT_FIAT=100` and calculate rolling skipped cumulative amount for the same `tx.invoice.external_id`, `tx.crypto`, and `tx.addr` over `AML_SKIP_CUMULATIVE_WINDOW_HOURS=24`. If the new cumulative amount is `<= AML_SKIP_CUMULATIVE_LIMIT_FIAT=300`, skip is allowed. If it would exceed the cumulative limit, skip is not allowed and the deposit must be AMLBot-checked unless coverage is unsupported/manual-review.

`build_skipped_check(tx)` must create an `AmlCheck` with `status="skipped"`, `deposit_decision="credit"`, `decision_reason="amount_below_aml_threshold"`, `score=None`, `skip_reason="amount_below_threshold"`, and populated `min_check_amount_fiat`, `cumulative_window`, `cumulative_amount_fiat`, and `cumulative_limit_fiat`.

`decision_from_provider_result(tx, result)` must implement: provider success with usable `score <= AML_MAX_ACCEPT_SCORE` resolves to `deposit_decision="credit"` and `decision_reason="score_below_threshold"`; score above threshold resolves to `manual_review` and `risk_score_above_threshold`; provider error resolves to `manual_review` and `aml_provider_error`; timeout resolves to `manual_review` and `aml_pending_timeout`; unsupported coverage resolves to `manual_review` and `unsupported_asset`; missing score resolves to `manual_review` and `incomplete_aml_result`.

Add `tests/test_aml_policy.py` for skip under threshold, repeated skipped deposits under cumulative limit, cumulative limit exceeded requiring check, skipped score is `None`, score equal threshold credits, above-threshold manual review, missing score manual review, unsupported asset manual review.

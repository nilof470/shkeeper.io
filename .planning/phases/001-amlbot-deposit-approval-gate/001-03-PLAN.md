Trust boundaries:
- Coin sidecar to SHKeeper walletnotify boundary.
- SHKeeper to aml-shkeeper sidecar boundary.
- SHKeeper to merchant callback boundary.

STRIDE register:
| ID | Threat | Component | Mitigation |
|----|--------|-----------|------------|
| T-03-01 | Callback is sent before AML terminal state | `walletnotify`, `send_callbacks` | Replace direct final send with AML processing and gate `send_notification`. |
| T-03-02 | Retry creates multiple provider checks | `aml_processing.py` | Reuse existing `AmlCheck` by `transaction_id` and stable `idempotency_key`. |
| T-03-03 | Merchant receives inconsistent retry payloads | `send_notification` | Build AML fields from persisted `AmlCheck` snapshot. |
| T-03-04 | Unconfirmed observability callback becomes creditable | unconfirmed callback path | Do not add `deposit_decision`, `decision_reason`, or `aml` to unconfirmed callbacks. |
</threat_model>

  <files>shkeeper/services/aml_processing.py, shkeeper/services/aml_policy.py, tests/test_aml_processing.py</files>
  <read_first>shkeeper/models.py, shkeeper/services/aml_policy.py, shkeeper/services/aml_shkeeper_client.py, .planning/phases/001-amlbot-deposit-approval-gate/001-CONTEXT.md</read_first>
  <action>
Create `shkeeper/services/aml_processing.py` with public functions `ensure_aml_for_transaction(tx)`, `process_pending_aml_checks(now=None)`, `refresh_aml_check(aml_check)`, and `is_callback_allowed(tx)`.

`ensure_aml_for_transaction(tx)` must:
1. Return existing `tx.aml_check` if present.
2. Resolve unsupported coverage into terminal `manual_review` with reason `unsupported_asset` or `limited_analysis_requires_review`.
3. Create a skipped terminal check if `should_skip_aml(tx)` allows skip.
4. Otherwise create a pending `AmlCheck` with `deposit_id=shkeeper-tx-{tx.id}`, `idempotency_key={tx.crypto}:{tx.txid}:shkeeper-tx-{tx.id}`, provider `"amlbot"`, coverage asset/network, and timeout fields.
5. Call `AmlShkeeperClient.create_check(...)` only for above-threshold or cumulative-limit-exceeded supported assets, then persist the normalized sidecar response snapshot.

`process_pending_aml_checks(now=None)` must select non-terminal `AmlCheck` rows with `next_retry_at is null or next_retry_at <= now`, call `AmlShkeeperClient.get_check(deposit_id)`, update local snapshot, and resolve terminal decisions through `decision_from_provider_result`. Checks past `timeout_at` must become `deposit_decision="manual_review"` and `decision_reason="aml_pending_timeout"`.

`is_callback_allowed(tx)` must return true for outgoing invoices or for incoming transactions whose `AmlCheck` is terminal. It must return false for pending/checking/missing AML on incoming transactions.

Add `tests/test_aml_processing.py` for: unsupported asset terminal manual_review, skip creates no sidecar call, supported above-threshold creates sidecar check, pending check blocks callback, timeout resolves manual_review, duplicate ensure reuses same `AmlCheck`.
  </action>
